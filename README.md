# 🐾 The Collaborative Cat Tale

Welcome to the Team Repository! This project is a hands-on exercise to practice **Git Collaboration** as outlined in Lecture 6.

## 📋 Project Instructions

Follow these steps to contribute to our story without breaking the "purr-fect" code.

### 1. Setup
* [cite_start]**Fork** this repository to your own GitHub account[cite: 132, 135].
* [cite_start]**Clone** your fork to your local machine: `git clone <your-ssh-url>`[cite: 96, 135].
* [cite_start]Verify your remote is set to **origin**: `git remote -v`[cite: 98, 135].

### 2. Feature Branching
* [cite_start]**Create a new branch** for your contribution: `git switch -c <your-name>`[cite: 136, 162].
* Add a short paragraph to `story.md`. [cite_start]Be creative![cite: 136, 162].
* [cite_start]**Stage and Commit** your changes: `git add .` and `git commit -m "Add cat plot twist"`[cite: 24, 125].

### 3. Collaboration & Pull Requests
* [cite_start]**Push** your branch: `git push --set-upstream origin <your-name>`[cite: 136, 162].
* [cite_start]Open a **Pull Request (PR)** on GitHub from your branch into `main`[cite: 154, 162].
* [cite_start]**Review** a teammate's PR: Leave a comment and approve it before merging[cite: 158, 162].

### 4. Keeping it Clean
* [cite_start]If your push is rejected, **Pull** the latest changes: `git pull`[cite: 122, 124].
* [cite_start]Use `git stash` if you need to pull while you have uncommitted work[cite: 116, 117].
* [cite_start]Use `git blame story.md` to see who wrote which part of the final tale!.

### 5. Final Release
* Once the story is complete, one team member will create a **tag**:
    `git tag -a submission -m "Final Story Release"`[cite: 166, 168].
    `git push --tags`[cite: 166, 170].

---
*Happy coding, and watch out for merge conflicts!* 🐱
